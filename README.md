## ðŸ”„ System Workflow
1. User submits a suspicious SMS or Email via **Microsoft Forms**
2. Data is stored in a **SharePoint List**
3. **Power Automate**:
   - Validates the submission  
   - Applies fraud detection rules  
   - Flags records as *Fraud* or *Safe*
4. Data is processed in **Excel**
5. **Power BI** visualizes fraud insights in real time

## âš™ï¸ Fraud Detection Logic
The system uses **rule-based detection**, including:
- High-risk sender email domains
- Suspicious message codes
- Repeated submissions from the same sender
- Abnormal activity frequency
- User feedback ratings

> Rules are configurable and can be enhanced without changing system architecture.
